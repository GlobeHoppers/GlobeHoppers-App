# GlobeHoppers v8.2.6 Work Package 4 Fast Conflict Hotfix

This hotfix replaces the retryable PostgreSQL serialization-failure code used for stale Edit Hop revisions. The revision check is now part of the trip UPDATE predicate and a stale revision raises a normal application exception immediately.

## Acceptance test

1. Open the same trip in two browser windows.
2. Save Window 1.
3. Save stale changes from Window 2.
4. Window 2 must promptly show: `This trip changed in another session. Reload before saving.`
5. The save button must return to `Save Hop` without waiting for the client timeout.
6. Window 1's saved data must remain unchanged.
