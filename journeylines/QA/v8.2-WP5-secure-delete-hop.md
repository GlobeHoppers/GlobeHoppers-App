# GlobeHoppers v8.2 Work Package 5 — Secure Delete Hop

- Private trip deletion uses `delete_private_trip` with `auth.uid()`, map permission checks, and `updated_at` concurrency protection.
- Child `trip_legs` and `trip_hoppers` rows are removed by database cascades.
- Timeline and globe reload only after Supabase confirms deletion.
- Delete confirmation, stale-edit conflicts, and retryable failures use GlobeHoppers-styled popups instead of browser alerts.
- Batch Add and timeline reordering remain outside this work package.
