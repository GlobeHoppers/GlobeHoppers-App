# GlobeHoppers v8.2 Work Package 1

## Scope

This work package adds read-only, account-specific Supabase loading. It deliberately does not connect Add/Edit/Delete Hop to Supabase yet.

## Expected behavior

- Signed-out users see an empty private-map welcome state rather than the bundled personal timeline when `VITE_ENABLE_CLOUD_TRAVEL_DATA=true`.
- After sign-in, the app loads the authenticated user's selected map, Hoppers, trips, legs, trip-Hopper links, and referenced locations.
- Account state is cleared immediately on sign-out or user change.
- Add/Edit/Delete and Hopper editing are blocked while `VITE_ENABLE_CLOUD_TRAVEL_WRITES=false`.
- The legacy JSON repository remains available when cloud travel data is disabled.

## Manual acceptance tests

1. Sign in as User A and confirm only User A's map records load.
2. Sign out and confirm routes, timeline rows, selected trip, and Hopper state clear.
3. Sign in as User B and confirm no User A records flash or remain.
4. Refresh while signed in and confirm the same account reloads.
5. Confirm Add Hop displays the read-only migration notice.
6. Confirm screensaver and globe rendering do not crash with an empty map.
7. Set `VITE_ENABLE_CLOUD_TRAVEL_DATA=false` and confirm the legacy JSON timeline still loads.
