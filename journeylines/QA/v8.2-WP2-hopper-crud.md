# GlobeHoppers v8.2 Work Package 2

## Scope

- Account-owned Hopper create, edit, reorder, and delete through the normal Supabase publishable key.
- Hoppers are always written to the authenticated account's selected map.
- Full trip Add/Edit/Delete remains disabled.
- HopSquads remain disabled in cloud mode because they do not yet have a database table.

## Preview flags

```text
VITE_ENABLE_CLOUD_TRAVEL_DATA=true
VITE_ENABLE_CLOUD_HOPPER_WRITES=true
VITE_ENABLE_CLOUD_TRAVEL_WRITES=false
```

## Acceptance

1. User A creates a Hopper; User B cannot see it.
2. User A edits the Hopper name/color; refresh preserves it.
3. User A deletes the Hopper; refresh confirms removal.
4. User B performs the same lifecycle independently.
5. Add/Edit/Delete Hop remains blocked.
